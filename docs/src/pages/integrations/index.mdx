ZUI ships the **markup and styling** — accessible, themeable components with no
behaviour baked in. That makes it a natural fit for **headless libraries** that
own state and logic but leave rendering to you.

These guides show how to wire ZUI's components up to popular headless tools.
The pattern is always the same: the library manages state, ZUI renders it.
